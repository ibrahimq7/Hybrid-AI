# Hybrid AI Assistant

Hybrid AI Assistant is a Flask chatbot that answers from a trusted FAQ dataset first, searches uploaded knowledge second, and uses Groq for fast AI-generated responses when local knowledge is not enough.

## Features

- Professional login and signup before chatbot access
- Hashed local password storage with Flask sessions
- Text questions and browser voice input
- Optional spoken replies using browser speech synthesis
- Strict mode for answers only from FAQ and uploaded knowledge
- Hybrid mode that uses Groq for non-strict questions
- FAQ matching from `faq_data.csv`
- RAG-style search over uploaded or ingested knowledge
- Knowledge ingestion for CSV, JSON, TXT, PDF, DOCX, manual text, and website URLs
- Persistent chat history in the browser
- Health and knowledge API endpoints for deployment checks

## How the Assistant Answers

1. It first checks `faq_data.csv` for a confident FAQ match.
2. It searches indexed uploaded knowledge for relevant chunks.
3. In strict mode, it answers only from trusted FAQ or uploaded knowledge.
4. In hybrid mode, it uses Groq with retrieved context for natural answers.
5. If Groq is unavailable, it explains that clearly instead of failing silently.

## Setup

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

Create a `.env` file in the project root:

```env
GROQ_API_KEY=your_groq_api_key_here
GROQ_MODEL=llama-3.1-8b-instant
GROQ_BASE_URL=https://api.groq.com/openai/v1
SESSION_SECRET=replace_with_a_long_random_secret
```

Run the app:

```bash
python app.py
```

Open:

```text
http://127.0.0.1:5000
```

## Deployment Notes

- The Flask app reads `PORT` automatically for deployment platforms.
- Users must log in before accessing the chatbot or protected API routes.
- Password hashes are stored in `data/users.json`, which is ignored by Git.
- Voice input uses the browser Web Speech API, so it works best in Chrome or Edge on `localhost` or HTTPS.
- Voice replies are generated by the browser, so no extra server TTS package is required.

## API Endpoints

- `GET /health` - service status
- `POST /api/ask` - ask a chatbot question
- `GET /api/knowledge` - knowledge base stats
- `POST /api/knowledge` - upload or add new knowledge

Example request:

```json
{
  "question": "What is AI?",
  "history": [],
  "mode": "hybrid"
}
```
